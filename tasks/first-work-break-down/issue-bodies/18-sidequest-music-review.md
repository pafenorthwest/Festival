## Summary

Build the music review app side quest so staff can normalize repertoire data into a reusable catalog with canonical composer/work references and reviewer workflows.

## Scope

- Create a review queue for submitted titles and composers
- Normalize repertoire records into canonical internal forms
- Store IMSLP/reference links where applicable
- Support reviewer ownership and flagging workflows
- Build the API and UI surfaces needed for internal review work

## Deliverables

- Catalog/review data model and APIs
- Reviewer queue and editing interface
- Flagging and assignment workflow for problematic repertoire entries
- Canonical composer/work reference storage

## Acceptance Criteria

- Reviewers can claim and process repertoire cleanup work
- Canonical composer/work data can be stored separately from raw submissions
- Flagged entries are easy to find and resolve
- The resulting catalog is reusable in later registration and review workflows

## Summary

Build the volunteer portal side quest so volunteers can self-serve into available roles and administrators can see coverage gaps in real time.

## Scope

- Model volunteers, roles, shifts, and assignments
- Support browse, sign-up, cancellation, and reminder flows
- Enforce shift capacity limits
- Provide admin visibility into open coverage

## Deliverables

- Volunteer scheduling data model and APIs
- Volunteer-facing signup/cancel experience
- Admin coverage dashboard
- Reminder automation hooks for upcoming shifts

## Acceptance Criteria

- Volunteers can self-assign to available shifts
- Shift capacity is enforced
- Admins can see coverage gaps without manual reconciliation
- Reminder notifications can be sent before the event

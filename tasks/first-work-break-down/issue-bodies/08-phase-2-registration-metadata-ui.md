## Summary

Build the Phase 2 frontend flows that capture and edit performer, instrument, repertoire, and related registration metadata against the entitlement model.

## Scope

- Create performer management screens tied to purchased entitlements
- Capture instrument, repertoire, age-group, and related registration metadata
- Support editing local metadata without re-entering Shopify checkout
- Reflect entitlement state and missing metadata in the UI

## Deliverables

- UI flows for entitlement-backed registration metadata entry
- Performer creation/edit forms and associated validation
- Status views showing which entitlements are ready for the next phase

## Acceptance Criteria

- Users can attach metadata to an entitlement after purchase
- Performer and repertoire data can be edited locally
- The UI clearly indicates incomplete registration metadata
- The flow aligns with the entitlement ownership model from Phase 2 backend APIs
